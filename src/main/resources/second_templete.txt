

UMLのクラスとオブジェクトを，json記法を用いてそれぞれ以下のような構造で表現する
{クラス名:{
属性名:型
}
}
{オブジェクト名:{
属性名:値
}
}

このとき，json記法を用いて以下のデータ構造が定義されている。

ScreenTransitionModel

このデータ構造に対して，以下のデータ具体値が定義されている．

ObjectDiagram

今，System を考えた際，「GenerateIntention」を達成した時のデータの具体値の変化を，
以下の条件群を守った上で回答せよ．

・具体値は日本で一般的に入力される値とし、セキュリティ上問題ない値とする
・同様のデータ構造が複数ある場合は，リスト表記は"絶対に"使用せず，名前の後ろに番号を付加する

また、データ群の出力はjson形式で表す。以下に出力形式の例を２つ示す。
出力例1:
・ユーザ情報オブジェクトを2つ作成する
変化前：
{
"ユーザ情報": {
"ユーザID": "文字列入力",
"ユーザ名": "文字列入力"
}}
変化後：
{
"ユーザ情報1": {
"ユーザID": "123456",
"ユーザ名": "佐藤一郎"
},
"ユーザ情報2": {
"ユーザID": "987654",
"ユーザ名": "田中太郎"
}}
出力例2:
・住所情報オブジェクトを1つ追加する
変化前:
{
"住所情報1":{
"都道府県":"東京",
"市区町村":"世田谷区"
}}
変化後：
{
"住所情報1":{
"都道府県":"東京",
"市区町村":"世田谷区"
},
"住所情報2":{
"都道府県":"大阪",
"市区町村":"大阪市"
}}

回答をデータとしてそのまま利用するため、余計な文言は答えず、jsonデータのみを回答せよ
